## **Overview**

The **Karl Insight Agent** requires access to several Azure and Microsoft services. To enable the Karl workload in a **guest tenant workspace**, a **Global Administrator** must grant admin consent and register the required service principals.

This guide provides clear, step-by-step instructions to verify the Karl application, register it if needed, and configure the required services.

## **Required Services**

The Karl Insight Agent depends on the following Azure and Microsoft services:

## **Step 1: Verify Karl Application in Guest Tenant**

1.  Sign in to the **Microsoft Entra Admin Center** ([Azure Portal)](https://portal.azure.com/).
2.  Navigate to: **Enterprise Applications → All Applications**
3.  Remove any filters and search for the **Karl application**.

## **Step 2: Verifying Karl Application Configuration**

Check that if Karl is auto populated under the **Enterprise Applications → All Applications**<br/>then proceed with next steps, in case the name does not appear follow the below instruction

<br/>Use **Azure Cloud Shell**

Connect-AzureAD

New-AzureADServicePrincipal -AppId "\< Karl\_Application\_client ID>"

After running the commands:

*   Return to **Enterprise Applications → All Applications**
*   Confirm that the Karl application now appears

## **Step 3: Add Required Service Principals**

after confirming that the Karl application is available, review the **Enterprise Applications → All Applications** section and identify any required services that are missing in your tenant. For any services that are not listed, follow the steps below to manually add the corresponding service principals.

### **Connect to Azure AD**

Connect-AzureAD

### **Create Service Principals**

Run the following command for **each service** listed below:

New-AzureADServicePrincipal -AppId "\<SERVICE\_APP\_ID>"

| Service | Command |
| --- | --- |
| Azure SQL Database | New-AzureADServicePrincipal -AppId "022907d3-0f1b-48f7-badc-1ba6abab6d66" |
| Azure Storage | New-AzureADServicePrincipal -AppId "e406a681-f3d4-42a8-90b6-c2b029497af1" |
| Azure Data Explorer | New-AzureADServicePrincipal -AppId "2746ea77-4702-4b45-80ca-3c97e680e8b7" |
| Power BI Service | New-AzureADServicePrincipal -AppId "00000009-0000-0000-c000-000000000000" |
| Microsoft Graph | New-AzureADServicePrincipal -AppId "00000003-0000-0000-c000-000000000000" |

## **Step 4: Adding workload**

After adding all service principals, grant tenant-wide admin consent.

Go to Fabric and try to add the Karl from the Workload Hub
