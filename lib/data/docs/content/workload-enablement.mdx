## Overview

This document describes the end-to-end process for enabling the **Karl workload** in Microsoft Fabric using the **Fabric Admin Portal**. The steps are written for business and technical stakeholders, with clear checkpoints and image placeholders.

## Prerequisites

*   You have **Fabric Admin** or equivalent permissions.
*   The **Karl workload** is already registered and visible in Fabric.
*   You have access to at least one **workspace** where Karl should be enabled.
*   Fabric Capacity Requirements for the Workspace – The workload can work on smaller capacities (F2 & F4), but for high concurrent usage we suggest at least F16 for responsiveness.

## User Flow

## Step 1: Open Fabric Admin Portal

*   Sign in to **Microsoft Fabric**.
*   Navigate to **Admin Portal**.
*   From the left navigation, select **Workloads**.

Click on the settings icon in the top right and click on Admin portal:

## Step 2: Search and Select Karl Workload

*   In the **Workloads** page, use the search bar.
*   Search for **Karl**.
*   Click on **Karl** from the list.

This action opens the **workload details pane** on the right side of the screen.

![A screenshot of a computer
AI-generated content may be incorrect.](/images/extracted/img_98dfd3e4.png)

## Step 3: Open Workload Details Pane

*   Review the workload information displayed in the right-side pane.
*   Click on **Add workload**.

![A screenshot of a computer
AI-generated content may be incorrect.](/images/extracted/img_9ddae637.png)

## Step 4: Select Target Workspace

*   After clicking **Add workload**, you will be prompted to choose a **workspace**.
*   Select the workspace(s) where Karl should be enabled.
*   Confirm your selection.

## Step 5: Microsoft Authentication

*   You will be redirected to **Microsoft authentication**.
*   Sign in using your organizational credentials.
*   Grant the required permissions when prompted.

## Step 6: Submit Workload Request

*   After successful authentication, a **request form** is displayed.
*   If there is an error while displaying the request form, It is a likely occurrence that you might have not added required azure services to your tenant. You can use the documentation provided [here](https://kanerika.sharepoint.com/:w:/s/FLEX/IQDYwVAeSxNDSKPzjt5oe6bIASHaueKsE2E7Fbl7wss-YLQ?e=DByWc9) to resolve the issue.
*   Enter the **reason for requesting the Karl workload** (business justification).
*   Click **Submit**.

![A screenshot of a computer screen
AI-generated content may be incorrect.](/images/extracted/img_2629c284.png)

## Step 7: Approval by Admin

*   The submission request is **sent to Admin for approval**.
*   Post submission:
*   The admin team receives the request (via internal notification/email – _confirmation pending_).
*   The request is reviewed and approved.

![A screenshot of a computer screen
AI-generated content may be incorrect.](/images/extracted/img_55664bb5.png)

## Step 8: Use Karl in Approved Workspace

*   Once approval is granted:
*   The selected workspace is enabled for the **Karl workload**.
*   Users with access to the workspace can now:
*   Add **Karl items**.
*   Start using Karl features as per granted permissions.

## Summary Flow

Admin Portal → Workloads → Search Karl → Select Karl → Add Workload → Choose Workspace → Microsoft Authentication → Submit Reason → Kanerika Approval → Karl Available in Workspace

## Notes & Best Practices

*   Provide a **clear and concise business justification** to avoid approval delays.
*   Ensure workspace owners are informed before enabling the workload.
*   Validate access post-approval by creating a sample Karl item in the provided workspace list.
